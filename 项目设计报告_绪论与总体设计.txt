《行动手帐》项目设计报告书

1 绪论

1.1 选题背景与意义

[选题背景]
在快节奏的现代生活中，个人效能管理与心理健康关注日益成为人们的刚需。大多数人面临着两个主要痛点：一是日常任务繁杂，容易导致执行力不足，急需高效的待办清单（Todo List）进行管理；二是缺乏对每日生活的深度复盘与反思，导致经验无法沉淀。目前市面上的工具大多功能单一，待办事项管理软件通常只关注“做没做”，而日记类软件只关注“记没记”。这种功能上的割裂，使得用户无法形成“行动-反馈-记录”的良性闭环，难以从日常行动中获得持续的成就感与成长动力。

[选题意义]
本项目旨在设计并实现一个名为“行动手帐（Action Log）”的Web应用，创造性地将“任务管理”与“心情日记”深度融合。

在用户价值方面，它通过可视化的进度反馈和结构化的记录方式，帮助用户提高行动力。用户在完成任务的同时，能够即时记录当下的心情与感悟，将枯燥的日常转化为有成就感的“个人成长史”，养成每日复盘的良好习惯。

在技术实践方面，本项目是一个典型的现代前端单页应用（SPA）。它不仅涵盖了基础的数据增删改查（CRUD），还涉及复杂的嵌套状态管理（日记包含任务列表）、组件间通信、动态交互动画、数据可视化图表以及响应式布局适配。通过本项目的开发，能够全面深入地实践 Vue.js 生态系统（Vue 3, Pinia, Vue Router）以及后端服务（Supabase）的综合应用，具有较高的工程实践价值。

1.2 国内外研究现状

在个人效能管理与数字化记录领域，国内外已涌现出大量成熟的解决方案，主要分为“纯任务管理”和“纯笔记/日记”两大流派。

[国外研究现状]
在国外，个人生产力工具市场发展较早且细分程度高。
以 Todoist、Microsoft To Do 为代表的纯任务管理工具，严格遵循 GTD (Getting Things Done) 理念，强调“收集-整理-执行”的流程，功能专注于清单的层级管理与协作，但在任务完成后的情感反馈与经验复盘方面较为薄弱。
另一方面，以 Day One、Journey 为代表的日记类应用，专注于富文本记录与多媒体回忆，强调情感的抒发，但往往与用户的日常行动计划脱节，缺乏对当日行为的结构化追踪。
近年来，Notion 等“All-in-One”类工具试图整合两者，通过强大的数据库功能实现高度自定义，但其较高的学习成本和复杂的配置过程，往往让普通用户望而却步，难以快速上手形成习惯。

[国内研究现状]
国内市场受移动互联网普及的影响，效率类应用呈现出“功能聚合”与“社交化”的趋势。
“滴答清单”是国内任务管理的佼佼者，它在任务管理的基础上融入了番茄钟、打卡习惯等功能，尝试引导用户关注执行过程，但其本质仍偏向于工具属性，缺乏对用户内心感受的记录维度。
同时，各类“手账App”（如Mori手账、时光手账）深受年轻群体喜爱，它们提供了丰富的贴纸、胶带等排版素材，极大地满足了用户的审美需求，但这类应用普遍存在“重形式轻内容”的问题，且操作繁琐，往往沦为一种单纯的装饰性记录，难以真正起到提升执行力的作用。

[总结与本项目切入点]
综上所述，当前市场缺乏一种既能轻量化管理每日行动，又能即时记录情感反馈的整合型工具。本项目的“行动手帐”正是基于这一现状，试图打破“清单”与“日记”的界限，通过“行动驱动复盘”的设计理念，填补市场空白。

2 网站总体设计

2.1 需求分析

2.1.1 功能需求
本系统主要面向希望提升个人执行力和养成复盘习惯的用户，核心功能需求如下：
1. 用户认证功能：支持用户的注册、登录及身份验证，确保用户数据的私密性与安全性。
2. 行动清单管理（Action List）：
   - 支持任务的快速创建、编辑和删除。
   - 支持任务状态标记（完成/未完成），完成后触发视觉反馈。
   - 支持任务优先级设置（高/中/低）及分类标签管理。
   - 支持通过拖拽方式调整任务排序。
3. 复盘日记系统（Review Journal）：
   - 提供文本编辑区域，用于撰写每日感想与总结。
   - 提供心情追踪功能，用户可选择当日心情图标。
   - 具备“封存”机制，即每日任务与日记完成后进行归档保存。
4. 数据仪表盘（Dashboard）：
   - 提供可视化的统计图表，展示近期任务完成率趋势。
   - 展示心情分布统计及累计打卡天数等激励数据。
5. 历史归档（Archive）：
   - 以时间轴或卡片流形式展示过往日记与任务记录。
   - 支持按日期、关键词或心情标签进行搜索与筛选。

2.1.2 非功能需求
1. 响应式设计：系统需适配桌面端（PC）和移动端设备，在不同屏幕尺寸下均能保持良好的布局与交互体验。
2. 交互体验：操作需流畅自然，关键操作（如添加、删除、完成任务）应配备平滑的过渡动画，提供即时的用户反馈。
3. 数据持久化：用户数据需实时保存至云端数据库，防止数据丢失，并支持多端同步。
4. 代码规范性：遵循现代前端工程化规范，代码结构清晰，具备良好的可维护性和扩展性。

2.1.3 资料收集
项目开发过程中收集了大量关于时间管理方法论（如番茄工作法、GTD）的资料，参考了市面上优秀待办应用（如TickTick、Notion）的UI/UX设计模式，并查阅了 Vue.js 3、Pinia、Supabase 等官方技术文档以确保技术实现的准确性。

2.2 功能模块设计

本网站主要由以下五大核心功能模块组成：

1. 用户认证模块
   - 功能：负责用户的注册、登录、注销及会话保持。
   - 结构：包含登录表单、注册表单、验证码校验组件。

2. 核心编辑模块（首页/今日视图）
   - 功能：这是用户的主要操作区域，集成了任务管理和日记编辑。
   - 结构：
     - 任务面板：包含任务输入框、任务列表项（含复选框、优先级标签、删除按钮）。
     - 日记面板：包含文本域、心情选择器、封存（保存）按钮。
     - 进度展示：实时计算当日任务完成百分比并显示进度条。

3. 数据统计模块（Dashboard）
   - 功能：对用户历史行为数据进行分析与展示。
   - 结构：集成 ECharts 图表组件，展示“任务完成率折线图”、“心情分布饼图”以及“核心指标卡片”（如连续打卡天数）。

4. 历史归档模块（Archive）
   - 功能：查看和检索过往的记录。
   - 结构：包含搜索筛选栏（日期/关键词）、历史记录卡片列表（展示日期、摘要、心情、任务概览）。

5. 个人中心与设置模块
   - 功能：管理用户个人信息及应用偏好设置。
   - 结构：包含头像昵称修改、主题切换（深色/浅色模式）、番茄钟设置等。

2.3 技术框架和开发环境

本项目经过深思熟虑的技术选型，构建了一套高效、现代且易于维护的开发体系。以下是各核心技术组件的详细解析：

[开发环境与工具]
- **Visual Studio Code (VS Code)**: 微软推出的开源、跨平台代码编辑器，是目前前端开发领域的首选工具。
  - **生态优势**: 依托其庞大的插件市场，本项目安装了 Volar (提供 Vue 3 语法高亮与智能补全)、ESLint (代码规范检查)、Prettier (代码格式化) 等插件，构建了强大的辅助开发环境。
  - **TypeScript 原生支持**: VS Code 对 TypeScript 拥有业界最顶级的原生支持，提供了精准的类型提示、定义跳转和即时重构功能，极大降低了类型错误的发生率。
  - **集成工作流**: 其内置的集成终端方便开发者直接运行 npm 脚本与 Git 命令，使得编码、调试、版本控制在同一界面内完成，大幅提升了开发效率。
- **Git 版本控制**: 采用 Git 进行源代码的版本管理，配合 GitHub/GitLab 进行远程托管，确保代码历史可追溯，便于分支管理和团队协作。

[前端核心技术栈]
- **Vue 3 (Composition API)**: 采用 Vue.js 的最新主版本。利用组合式 API (Composition API) 将相关的业务逻辑（如日记的增删改查）聚合在一起，解决了传统 Options API 导致的代码逻辑分散问题，提高了代码的可读性与复用性。
- **TypeScript**: 作为 JavaScript 的超集，TypeScript 在编译阶段即可拦截空指针、类型不匹配等常见错误。本项目通过定义 User, JournalEntry, Task 等严谨的接口（Interfaces），确保了数据流转的类型安全。
- **Vue Router 4**: 负责单页应用（SPA）的路由管理。利用其“导航守卫”特性，实现了对未登录用户的拦截与权限控制，保护用户隐私页面。
- **Pinia**: 作为 Vue 官方推荐的下一代状态管理库，Pinia 相比 Vuex 更加轻量化。它去除了 mutations 的繁琐概念，提供了符合直觉的 API 设计，并能完美推断 TypeScript 类型，用于高效管理用户会话和全局应用状态。

[数据持久化与云端服务 (Supabase)]
本项目选用 Supabase 作为后端即服务 (BaaS) 平台，它是 Google Firebase 的开源替代方案，为前端开发者提供了完整的后端能力：
- **PostgreSQL 数据库**: Supabase 底层基于世界上最强大的开源关系型数据库 PostgreSQL。相比 NoSQL 方案，它能更好地处理复杂的关联数据（如用户与日记的一对多关系）。
- **Row Level Security (RLS)**: 安全是重中之重。我们利用 PostgreSQL 的行级安全策略 (RLS)，在数据库层面编写规则，强制确保“用户只能查看到自己创建的数据”，从根源上杜绝了数据越权访问风险。
- **Auth & Storage**: 直接复用 Supabase 提供的身份认证模块（处理注册、登录、Token刷新）和对象存储服务（用于存储用户上传的头像图片），极大缩短了开发周期。
- **离线优先**: 在前端层面，结合 LocalStorage 进行本地缓存，确保用户在网络不稳定时仍能正常浏览和编辑内容，待网络恢复后再与 Supabase 同步。

[构建与工程化]
- **Vite**: 选用基于 ESM 的新一代构建工具。Vite 利用现代浏览器的原生模块解析能力，实现了毫秒级的冷启动和热更新 (HMR)，彻底解决了传统 Webpack 项目启动慢的痛点。
- **Sass (SCSS)**: 使用 SCSS 预处理器编写样式，利用变量管理主题色，利用嵌套规则维护组件样式，配合 Flexbox 和 Grid 布局技术，实现了复杂的响应式界面。
- **Vitest + Vue Test Utils**: 建立了完善的单元测试体系，针对核心的业务逻辑函数和通用 UI 组件编写测试用例，确保重构或迭代时功能的稳定性。

[运行环境]
- **开发环境**: 依赖 Node.js 18+ 运行时环境。
- **生产环境**: 支持所有兼容 ES2020+ 标准的现代浏览器（Chrome, Firefox, Safari, Edge），并通过 CSS 媒体查询实现了移动端与桌面端的完美适配。

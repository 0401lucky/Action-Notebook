# 《行动手账 (Action Log)》Web应用开发项目计划书

- **项目名称：** 行动手账 (Action Log) —— 基于Vue.js的任务驱动型日志系统
- **项目类型：** Web前端开发 / 响应式单页应用 (SPA)
- **技术栈：** Vue.js (Vue 3) + Vue Router + Pinia/Vuex + CSS3 (Flex/Grid)
- **适用课程：** Web开发技术课程设计 / 前端框架应用

---

## 1. 项目背景与主要意义

### 1.1 项目背景

在快节奏的现代生活中，人们常面临两个问题：
1. 任务繁杂导致执行力不足（需要Todo列表）
2. 缺乏对每日生活的复盘与反思（需要日记）

市面上现有的工具往往功能单一，要么是纯粹的清单管理，要么是纯文本的日记记录，两者割裂，无法形成"行动-反馈-记录"的良性闭环。

### 1.2 项目意义

本项目旨在设计并实现一个名为"行动手账"的Web应用，创造性地将"待办事项管理（RPG任务化）"与"每日心情日记"深度融合。

- **用户价值：** 通过可视化的进度反馈和结构化的记录方式，帮助用户提高行动力，养成每日复盘的习惯，将枯燥的日常转化为有成就感的"个人成长史"。

- **技术实践价值（考核核心）：** 本项目是一个典型的现代前端单页应用（SPA）。它不仅涵盖了基础的增删改查（CRUD），还涉及复杂的嵌套状态管理（日记包含任务列表）、组件间通信、动态交互动画以及响应式布局适配。这能够全面展现开发者对 Vue.js 生态系统的掌握程度。

## 2. 项目目标与核心功能

### 2.1 总体目标

开发一个界面简洁现代、交互流畅、且具备多端适配能力的响应式Web应用。用户可以在一个统一的界面中规划当日行动、追踪完成进度，并结合此刻心情撰写复盘日志，最终形成结构化的每日记录。

### 2.2 核心功能模块

根据之前的规划，项目包含三大核心模块：

#### 模块一：核心编辑区（首页/今日视图）

这是应用的主战场，采用响应式布局（PC端双栏并列，移动端上下堆叠）。

**行动清单 (Action List Panel)：**
- **动态CRUD：** 支持快速添加新任务、删除错误任务。
- **状态交互：** 点击勾选完成任务，触发划线及完成特效动画。
- **进度可视化：** 通过计算属性实时统计当日完成度（如：70%），并以动态进度条展示。
- **任务优先级：** 支持为任务设置优先级（高/中/低），高优先级任务置顶显示并以醒目颜色标识。
- **分类标签：** 支持自定义标签（如：学习、工作、生活），便于任务归类管理。
- **拖拽排序：** 支持拖拽调整任务顺序，使用 `vuedraggable` 实现流畅的拖拽交互体验。

**复盘日记 (Review Journal Panel)：**
- **富文本记录：** 提供多行文本域用于撰写当日感想。
- **心情追踪：** 提供一组心情图标（开心、平淡、沮丧等）供选择绑定。
- **表单验证与封存：** 只有当日核心任务完成或日记字数达标后，才允许点击"封存今日"按钮，保存当日数据。

#### 模块二：数据仪表盘与历史回顾

**数据统计面板 (Statistics Dashboard)：**
- **完成率趋势图：** 以折线图展示近7天/30天的任务完成率变化趋势。
- **心情分布图：** 以饼图或环形图展示一段时间内的心情分布比例。
- **任务统计：** 显示累计完成任务数、连续打卡天数等激励性数据。
- **标签分析：** 统计各分类标签下的任务数量和完成情况。

> 技术实现：使用 ECharts 或 Chart.js 实现数据可视化图表。

**历史归档 (Archive View)：**
- 以时间轴或卡片流的形式展示过往的"手账"。
- 点击卡片可路由跳转至详情页查看当日完整的任务和日记。
- **搜索与筛选：** 支持按日期范围、心情类型、关键词搜索历史手账记录。

#### 模块三：全局交互与体验

- **响应式设计：** 利用 CSS Media Queries 和 Flex/Grid 布局，确保应用在桌面端、平板和手机端均有良好的展示效果。
- **过渡动画：** 在任务添加/删除、页面路由切换时加入平滑的 Vue Transition 动画，提升用户体验。
- **主题切换：** 支持浅色/深色模式切换，使用 CSS 变量实现主题色统一管理。
- **数据导出：** 支持将手账数据导出为 JSON 或 Markdown 格式，方便备份与迁移。

## 3. 技术架构与工具栈

为满足大作业对技术深度的要求，本项目选用主流且现代的技术栈：

| 分类 | 技术/工具 | 说明 (考核点对应) |
|------|-----------|-------------------|
| 核心框架 | Vue.js (Vue 3 Composition API) | 现代前端开发范式，体现对最新技术的掌握。 |
| 路由管理 | Vue Router 4.x | 实现单页应用的页面跳转（首页、历史页、详情页）。 |
| 状态管理 | Pinia (或 Vuex 4.x) | **核心难点。** 用于管理全局状态（当前的日记对象及其嵌套的任务数组），解决组件通信问题。 |
| UI与布局 | CSS3 (SCSS), Flexbox, CSS Grid | 实现复杂的响应式布局和自定义UI样式。不依赖重型UI框架，体现CSS基本功。 |
| 构建工具 | Vite | 高效的开发服务器和构建工具。 |
| 数据模拟 | LocalStorage / JSON Server | 前期使用本地存储实现数据持久化，后期可引入JSON Server模拟异步API请求。 |
| 数据可视化 | ECharts / Chart.js | 实现统计面板的图表展示（折线图、饼图等）。 |
| 拖拽交互 | vuedraggable | 实现任务列表的拖拽排序功能。 |

## 4. 项目实施进度计划 (建议周期：4-5周)

此计划按照标准的软件开发生命周期制定，确保按时高质量交付。

### 第一周：需求分析与框架搭建

- [ ] 明确功能细节，完成最终UI原型设计稿（参考已确认的UI图）。
- [ ] 初始化 Vue3 + Vite 项目环境。
- [ ] 配置 Vue Router，规划好路由结构（`/home`, `/archive`, `/detail/:id`）。
- [ ] 搭建基础页面结构 (HTML)，划分组件层级（如 `TodoList.vue`, `DiaryEditor.vue`, `ProgressBar.vue`）。

### 第二周：核心UI实现与静态页面开发

- [ ] 使用 CSS (Flex/Grid) 实现PC端和移动端的响应式布局。
- [ ] 开发基础UI组件：美化复选框、进度条样式、心情选择器组件。
- [ ] 确保静态页面在不同设备上的视觉还原度。

### 第三周：核心逻辑开发 (攻坚阶段)

- [ ] **配置状态管理 (Pinia/Vuex)：** 设计 Store 的数据结构，定义好 State (今日数据对象), Actions (添加任务, 勾选任务, 更新日记内容)。
- [ ] **实现 CRUD 逻辑：** 将 Todo 组件与 Store 绑定，实现任务的增删改功能。
- [ ] **实现计算逻辑：** 利用计算属性 (Computed) 实时计算任务完成百分比，驱动进度条更新。
- [ ] **实现表单逻辑：** 绑定日记输入框和心情选择器的数据。

### 第四周：交互增强与数据持久化

- [ ] **添加动画效果：** 为任务列表的增删添加 `<TransitionGroup>` 动画，为页面切换添加路由过渡动画。
- [ ] **实现数据存储：** 编写 Service 层，将 Store 中的数据保存到 LocalStorage，并在应用启动时读取。
- [ ] **表单验证：** 添加"封存"前的校验逻辑（如：任务未完成时给予提示）。

### 第五周：测试、优化与文档撰写

- [ ] **多端测试：** 在手机和PC浏览器上进行全面测试，修复样式和逻辑Bug。
- [ ] **代码优化：** 清理冗余代码，提取公共组件。
- [ ] **文档撰写：** 编写项目报告（可基于本计划书扩展），录制功能演示视频（大作业必备）。
